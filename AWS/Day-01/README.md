## 📘 Day 1: Cloud Computing Fundamentals & AWS Overview

### ☁️ What is Cloud Computing
Cloud computing is the delivery of computing services over the internet instead of owning physical hardware.

### 🔄 Evolution
Traditional IT → Virtualization → Cloud (AWS launched in 2006)

### 🆚 Traditional vs Cloud
- Traditional: High cost, slow, manual
- Cloud: Pay-as-you-go, fast, scalable

### 🚀 Benefits
- Cost reduction
- Scalability
- Global access
- Faster deployment

### 🌐 Cloud Deployment Models
- Public Cloud : Shared infrastructure (e.g., AWS)
- Private Cloud : Dedicated to one organization
- Hybrid Cloud : Combination of public + private

### 🧩 Service Models
- IaaS : Virtual servers, storage, networking (you manage OS & apps)
- PaaS : Platform to deploy apps (no OS management)
- SaaS : Ready-to-use software (e.g., Gmail, Google Docs)

### ⚙️ Core Principles
- Elasticity : Auto scale up/down
- Scalability : Handle growth in traffic
- High Availability : System stays online
- Fault Tolerance : Continues working despite failures

### 🔐 Shared Responsibility
- Cloud Provider: Infrastructure
- User: Data & applications

### ☁️ Providers
AWS, Azure, GCP, Oracle

### ⭐ Why AWS
- Global presence
- Scalable services
- Strong ecosystem

### 🌍 Infrastructure
- Regions
- Availability Zones
